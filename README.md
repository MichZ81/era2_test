# ERA2 Frontend

## Очередь генераций

Экран очереди доступен по маршруту `/queue`. Страница `QueuePage` остается тонкой композицией и рендерит виджет `GenerationQueue`, а вся логика очереди живет внутри `features/generation-queue`.

## Мок-движок

Бэкенд для очереди не используется. Обработка задач эмулируется на клиенте в `features/generation-queue/model/queueEngine.ts`:

- одновременно выполняется не больше двух задач;
- следующие задачи берутся из `queued` по FIFO;
- прогресс `running` задач обновляется таймерами;
- часть задач может завершиться ошибкой с пользовательским сообщением;
- при отмене задачи ее таймер очищается.

## Персистентность

Состояние очереди сохраняется в `localStorage` под ключом `era2_generation_queue` и восстанавливается при перезагрузке.

При восстановлении `running` задачи остаются в статусе `running`: после restore мок-движок заново синхронизируется с состоянием и продолжает тики прогресса. Мы не переводим такие задачи обратно в `queued`, чтобы глобальный статусбар и страница очереди продолжали показывать активную работу без сброса прогресса.

## Глобальный статусбар

`GenerationQueueStatusBar` монтируется на уровне приложения внутри общего `QueueProvider`, поэтому статусбар и страница очереди читают один источник правды.

Индикатор показывается только при наличии активных задач (`running + queued`):

- 1 активная задача: компактная карточка одной генерации;
- несколько активных задач: раскрытый статусбар с мини-списком;
- collapsed-состояние разворачивается по клику.

Кнопка или ссылка `Открыть очередь` ведет на `/queue`.

## Поиск и фильтры

Поиск по очереди находится в `QueueToolbar`. Пользовательский ввод сохраняется в query состояния очереди, а пересчет списка выполняется с debounce в `QueueProvider`.

Фильтрация и сортировка реализованы в `features/generation-queue/model/selectors.ts`.
